A novel mobile first flexbox BEM css grid.

Get started

Synopsis

ZEUS takes a new approach to the standard run-of-the-mill css grid. Firstly, it follows the BEM methodology, so right off the bat you've got a framework that fits into your large production applications. Secondly, it has a novel approach to mobile first development. By default, all layouts, rows and columns are in their barest form (no display, gutters, edges etc) and need to be selectively structured when the browser's viewport reaches specific minimum widths — think of it as progressive enhancement.

The default grid

It comes with 12 columns, a 75rem container (based on a 16px base font size) and 2rem gutters.

All devices

Designed for mobile first development, ZEUS uses the following default breakpoints to suit a range of screen-sizes and devices commonly used to browse the web.

XS: For screen widths less than 544px

.Layout__column--xs-*

SM: For screen widths 544px – 767px

.Layout__column--sm-*

MD: For screen widths 768px — 991px

.Layout__column--md-*

LG: For screen widths 992px – 1199px

.Layout__column--lg-*

XL: For screen widths 1200px and above

.Layout__column--xl-*

Documentation

Anywhere you see $ in the examples below, you're able to specify a breakpoint such as xs-, md-, lg- etc or leave it out completely. For example, Layout--$edges can be used as Layout--xs-edges or simply Layout--edges.

Layout

.Layout, .Layout--solid, .Layout--fluid, .Layout--edges, .Layout--breakaway

Every grid system needs a layout, it's the container that holds everything together. You can either use "Layout Layout--solid" for a centered container or "Layout Layout--fluid" for a container that spans the width of the whole page.

"Layout--$edges" adds edges to your layout (to stop text hitting the edge of the window)

"Layout--$breakaway" adds an extra gutter's worth of space under the layout to help it break away from content below it.

Row

.Layout__row, .Layout__row--gutters, .Layout__row--bottoms, .Layout__row--breaks, .Layout__row--direction-*, .Layout__row--valign-*

The next element of the grid system is the row. The row, by default is simply a container to hold columns. Using the available modifiers, you can change how the columns inside of it behave.

".Layout__row--$gutters" will add gutters the left and right of the columns within it.

".Layout__row--$bottoms" will add gutters the bottom of the columns within it.

".Layout__row--$breaks" will add a gutter below the row.

".Layout__row--direction-*" will change the direction in which columns are laid out. The options are row, row-reverse, column and column-reverse.

".Layout__row--valign-*" will change the vertical alignment of columns. The options are baseline, center, top, bottom and stretch.

Column

.Layout__column, .Layout__column--visible, .Layout__column--hidden, .Layout__column--*, .Layout__column--offset-*, .Layout__column--text-*

Columns hold your site content. They're ultimately what dictate the look and feel of your entire site and help reflow content across various viewport sizes.

".Layout__column--$visible" and ".Layout__column--$hidden" will respectively show and hide the column.

".Layout__column--$*" defines how wide the column will be. By default the options are 1-12, eg: ".Layout__column--6" or ".Layout__column--xs-4".

".Layout__column--$offset-*" defines the offset of a column, this follows the same options as above.

".Layout__column--$text-*" defines the text alignment for the column. The options are left, right, center and justify.

Example

Let's throw together a simple 3 column layout to get you started.

<div class="Layout Layout--solid Layout--edges">
    <div class="Layout__row Layout__row--bottoms Layout__row--gutters Layout__row--breaks">
        <div class="Layout__column Layout__column--visible">
            <p>Nullam quis risus eget urna mollis ornare vel eu leo.</p>
        </div>
        <div class="Layout__column Layout__column--visible">
            <p>Etiam porta sem malesuada magna mollis euismod.</p>
        </div>
        <div class="Layout__column Layout__column--visible">
            <p>Sed posuere consectetur est at lobortis.</p>
        </div>
    </div>
</div>

Verbosity?

We're all programmers here, and inherently we don't like to type more code than we have too. If you're worried about the verbosity, don't stress.

We've created a handy quick class to get you up and running faster for each of the primary elements (layout, row and column).

Here's an example that produces the same result as above.

<div class="Layout-quick">
    <div class="Layout__row-quick">
        <div class="Layout__column-quick">
            <p>Nullam quis risus eget urna mollis ornare vel eu leo.</p>
        </div>
        <div class="Layout__column-quick">
            <p>Etiam porta sem malesuada magna mollis euismod.</p>
        </div>
        <div class="Layout__column-quick">
            <p>Sed posuere consectetur est at lobortis.</p>
        </div>
    </div>
</div>

Off course you can always stick to the first method, for finer grain control.

Customization

Need more columns, different gutter sizes, a larger or smaller layout... or don't like your BEM classes starting with a capital letter? We've got you covered.

Simply check out src/common/variables to see what's available to tweak.
