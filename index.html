Synopsis

ZEUS takes a new approach to the standard run-of-the-mill css grid. Firstly, it follows the BEM naming methodology. Secondly, it has a novel approach to mobile first development. By default, all layouts, rows and columns are in their barest form (no display, gutters, edges etc) and need to be selectively structured when the viewport reaches specificied minimum widths — think of it as progressive enhancement.

The default grid

All devices

Designed for mobile first development, ZEUS uses the following default breakpoints to suit a range of screen-sizes and devices, inspired by Bootstrap 4.

XS: For screen widths less than 544px

.Layout__column--xs-*

SM: For screen widths 544px – 767px

.Layout__column--sm-*

MD: For screen widths 768px — 991px

.Layout__column--md-*

LG: For screen widths 992px – 1199px

.Layout__column--lg-*

XL: For screen widths 1200px and above

.Layout__column--xl-*
