Settings

When using the SASS source for Zeus, you can alter a number of variables to modify the grid to your needs.

Make sure you define the new values for the following variables before importing zeus.scss in your sass project.

Variable name Purpose Default
$container-width How wide the Layout container is in REM's when used without the Layout--fluid modifier. 75
$gutter-width How much space is defined around each column in REM's when used without the Layout__row--gapless modifier. 0.65
$column-count How many columns will be generated and available via column size modifiers. 12
$lc-blocks Whether lower case block names should be generated during compilation. Eg: Layout__row vs. layout__row. false

Container

  • Layout
  • --fluid
  • --hero

The container (otherwise known as the Layout) holds it all together. It can have a specific width, or it may be fluid - the decision is yours.

A standard container is simply a div with a class of Layout, while a fluid container has the classes Layout Layout--fluid.

<div class="Layout">
    <!-- ... -->
</div>

The container width is specified by the $container-width variable, which by default is set to 75rem.

Layout also has a special modifier named Layout--hero that creates a 100% viewport height container with all columns centered vertically within it. Check out the demo here if you're interested.

Row

  • Layout__row
  • --gapless
  • --wrap
  • --valign-top
  • --valign-bottom
  • --valign-center
  • --valign-baseline
  • --valign-stretch

Rows are exactly what you'd expect. The two most important modifiers you'll likely use are Layout__row--gapless which creates rows that contain columns without gutters, and Layout__row--wrap which allows rows to contain more columns that they'd naturally fit horizontally.

Column

  • Layout__column
  • --[%-]#
  • --[%-]offset-#
  • --[%-]quater
  • --[%-]three-quaters
  • --[%-]third
  • --[%-]two-thirds
  • --[%-]half
  • --[%-]text-left
  • --[%-]text-right
  • --[%-]text-center
  • --[%-]text-justify
  • [%-] is the optional viewport size, xs-, sm-, md-, lg- or xl-.
  • # is the column size, eg: 1, 2, 6, 12.
  • --hidden-%
  • % is the viewport size, xs, sm, md, lg or xl.
  • --fill-parent
  • --fill-child

Columns are where all the fun happens. We already have an example of a basic column setup in the row documentation above, so let's try something a bit more interesting. First of all, because all columns are built upon flexbox, we can add as many generic columns as we want, and they'll just fit themselves! Try it out...

If you don't want your columns automatically being sized, you can specify their sizes using the modifiers listed above. For example, using the standard 12 column grid, you could define columns such as Layout__column Layout__column--xs-6 or Layout__column Layout__column--xl-8.

Nesting columns is easy too, take a look at the example below.

<div class="Layout">
    <div class="Layout__row">
        <div class="Layout__column Layout__column--6">
            <div class="Layout__row">
                <div class="Layout__column Layout__column--6">
                    <p>child-6</p>
                </div>
                <div class="Layout__column Layout__column--6">
                    <p>child-6</p>
                </div>
            </div>
        </div>
        <div class="Layout__column Layout__column--6">
            <p>6</p>
        </div>
    </div>
</div>

If you want something more readable for simplistic column types, you can use the named modifiers such as Layout__column--third or Layout__column--md-half (which would only take up half the row at the md viewport).

Metro

  • Layout__column
  • --fill-parent
  • --fill-child

One additional benefit of using flexbox is it's extremely easy to create metro style grids using Zeus. All you need to do is use the modifiers Layout__row--valign-stretch Layout__column--fill-parent and Layout__column--fill-child inside of a standard grid setup.

<div class="Layout__row Layout__row--wrap Layout__row--valign-stretch">
    <div class="Layout__column Layout__column--4">
        <div class="Layout__row Layout__row--wrap">
            <div class="Layout__column Layout__column--12">
                <p class="MetroTile">Metro!</p>
            </div>
            <div class="Layout__column Layout__column--12">
                <p class="MetroTile">Metro!</p>
            </div>
        </div>
    </div>
    <div class="Layout__column Layout__column--fill-parent Layout__column--8">
        <p class="MetroTile Layout__column--fill-child">Metro!</p>
    </div>
    <div class="Layout__column Layout__column--8">
        <div class="Layout__row Layout__row--wrap">
            <div class="Layout__column Layout__column--6">
                <p class="MetroTile">Metro!</p>
            </div>
            <div class="Layout__column Layout__column--6">
                <p class="MetroTile">Metro!</p>
            </div>
            <div class="Layout__column Layout__column--12">
                <p class="MetroTile">Metro!</p>
            </div>
        </div>
    </div>
    <div class="Layout__column Layout__column--fill-parent Layout__column--4">
        <p class="MetroTile Layout__column--fill-child">Metro!</p>
    </div>
</div>

As you probably noticed in the example, we added the Layout__column--fill-child on our p tag. All that modifier does is set the css property flex: 1; - that's it! The rest is taken care of by Zeus.
